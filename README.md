# p0-w3-d2-t9 · Оформление заказа (большая задача)

> Тип: большой синтез JS-дней 1–2 · Репозиторий: один отдельный · Объёмнее обычных

> Подключить файл, отсмотреть в браузере.

## Задача

Собрать маленький, но цельный модуль оформления заказа в интернет-магазине. Он принимает данные
клиента и корзину, проверяет их, считает итог со скидками и собирает текст чека. Здесь сходится
всё за два дня JS: типы, условия, `switch`, `??` и truthy/falsy из первого дня — и функции,
параметры, значения по умолчанию, замыкания из второго. Кода заметно больше обычной задачи,
потому что это несколько связанных функций, работающих вместе.

```
оформление заказа — всё через функции
┌──────────────────────────────────────────────────────┐
│ 1. проверить данные клиента  → функции-валидаторы    │
│ 2. посчитать стоимость       → calcTotal(...)        │
│ 3. применить промокод        → ?? + скидка           │
│ 4. собрать чек строкой       → функции-форматтеры    │
└──────────────────────────────────────────────────────┘
вводные сверху файла, каждый шаг — вызов функции
```

## Вводные

В начале `checkout.js` задать переменными данные заказа (значения подбери сама, включая
заведомо плохие для проверки веток):

- клиент: имя, email, возраст;
- товар: название, цена за штуку, количество;
- тип доставки: `'pickup'`, `'courier'` или `'post'`;
- промокод (может отсутствовать).

## Что нужно сделать (всё через функции)

- **Проверка данных** — отдельные функции-валидаторы (можно переиспользовать из t4): имя
  заполнено, email с `@`, возраст — число. Если данные не проходят — заказ не оформляется,
  и видно, что именно не так.
- **Стоимость товара** — функция `calcTotal(price, qty)` возвращает сумму за товар; защититься
  от нечисловых/отрицательных значений.
- **Доставка** — функция, которая по типу доставки (`switch`) возвращает её стоимость
  (`pickup` — бесплатно и т.д.); неизвестный тип обработать.
- **Промокод** — функция применения скидки: если код не передан, взять «без скидки» через `??`;
  известный код (например `'SALE10'`) даёт −10%.
- **Скидка постоянным/льготным** — через тернарник или ветвление (например по возрасту).
- **Чек** — функции-форматтеры собирают итоговый текст: строка с товаром, доставкой, скидкой и
  финальной суммой (с разделением разрядов и валютой).
- Прогнать модуль минимум на 3 разных заказах (обычный, с промокодом, с плохими данными) и
  вывести результат каждого.

## Замыкание (обязательная часть)

Сделать фабрику `makeDiscount(percent)` (замыкание из t7): она возвращает функцию, применяющую
запомненный процент скидки к сумме. Создать из неё, например, `vipDiscount` и `promoDiscount`
и использовать в расчёте.

## Критерии приёмки

* Логика разнесена по функциям (проверка, стоимость, доставка, промокод, форматирование), а не
  один сплошной кусок; функции вызываются по имени и переиспользуются.
* Использованы темы обоих дней: `switch`, `??`, тернарник, проверка типов (день 1) и функции с
  параметрами/дефолтами + замыкание `makeDiscount` (день 2).
* На заведомо плохих данных заказ корректно отклоняется с понятной причиной; итоговая сумма
  никогда не `NaN` и не отрицательная.
* Модуль отработал на нескольких разных заказах, по каждому виден собранный чек.
